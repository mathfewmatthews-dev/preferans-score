# Пуля преферанса для Windows

Небольшое WinForms-приложение для записи и расчёта очков преферанса.

Также добавлена веб-версия с минималистичным интерфейсом:

```text
index.html
```

Её можно открыть прямо в браузере.

Поддержано:

- конвенции `Сочи` и `Питер`;
- от 3 до 5 игроков;
- настройка размера пули перед началом партии;
- визуальный лист пули с секторами, который перестраивается под 3, 4 или 5 игроков;
- типы записей: `Взятки`, `Распасы`, `Мизер`, `Ручной ввод`;
- роли защитников: `Пас`, `Вист`, `Полвиста`, `Отдыхает`;
- автоматический расчёт пули, горы, вистов и итогового баланса в вистах после каждой записанной раздачи;
- отметка игроков, закрывших пулю, и статус закрытия партии;
- сохранение и загрузка партии в JSON;
- копирование отчёта в буфер обмена.

## Сборка

На Windows запустите:

```bat
build-windows.bat
```

Готовый файл появится здесь:

```text
dist\PreferansCounter.exe
```

Проект не требует .NET SDK. Для сборки используется системный компилятор .NET Framework `csc.exe`, который обычно уже есть в Windows.

## Веб-версия

Откройте `index.html` в браузере. Веб-приложение работает без сервера и внешних зависимостей.

В веб-версии есть:

- стильная зелёная шапка с канделябром;
- минималистичный лист пули с секторами игроков;
- настройка Сочи/Питер, размера пули и 3-5 игроков;
- автоматический пересчёт после каждого хода;
- сохранение и загрузка партии в JSON;
- копирование отчёта.

## Принятые правила расчёта

Базовая стоимость игр:

- 6: 2
- 7: 4
- 8: 6
- 9: 8
- 10 и мизер: 10

В `Сочи` стоимость игры, штрафы и висты считаются по базовой таблице.

В `Питере` штрафы игрока, висты за взятки и распасы считаются в двойном размере; единица пули при итоговом пересчёте считается как 20 вистов.

Некоторые детали преферанса часто являются договорными, поэтому в приложении есть ручной ввод для корректировок.

## Работа с листом пули

На экране новой партии укажите количество игроков и размер пули, например `20`.

После нажатия `Записать ход` приложение сразу пересчитывает таблицу, журнал и визуальный лист пули. В секторе игрока показываются пуля, гора, написанные висты и текущий итог в вистах. Когда игрок достигает заданного размера пули, сектор помечается как `закрыл`; когда цель достигли все игроки, в статусе появляется `Пуля закрыта`. Дальнейший ввод остаётся доступным для доигрывания и ручных корректировок.
